LIFE-CYCLE METHODS:-

LIFE CYCLE are used in class component only.

In LIFE-CYCLE methods are classified into three types

  1. Mounting
  2. Updating
  3. Unmounting

MOUNTING:- When an instance of a component is being created and inserted into the DOM

Mounting is classified into four methods

  1. Constructor:-
    • A special type of function that will get called whenever a new component is created.
    • Used to initialize states & Binding events.
    • super():- it is used to call the constructor of it's parent class.
    • Not perform,http req.
  2. static getDerivedStatesFromProps:-(rarely used )
    • When the state of component depends on change in props.
    • set the state.
    • Not perform,http req.
  3. render:-
    • Invoked immediately after a component and it's child components have been rendered to DOM
    • Not perform,http req
  4. componentDidMount:-
    • Invoked immediately after a component and it's child components have been rendered to DOM
    • perform any AJAX call to load data.

UPDATING:- When a component is being re-render as a result of changes to either it's props or state.

Updating is classified into five methods

  1. static getDerivedStateFromProps :-
    • method is called every time a component is re-rendered.
    • set the state.
  2. sholudComponentUpdate :-
    • Dictates if the component should re-render or not.
    • Performance Optimization.
    • Not perform,http req
  3. render:-
    • Invoked immediately after a component and it's child components have been rendered to DOM
    • Not perform,http req
  4. getSnapshotBeforeUpdate :- (rarely used)
    • Called right before the changes from the virtual DOM are to be reflected in the DOM.
    • Capture some information from DOM.
  5. componentDidUpdate :-
    • Called after the render is finished in the re-render cycles.
    • Cause side effects.

UNMOUNTING:- When a component is being removed from the DOM.

Unmounting has only one method.

  1. componentWillUnmount:-
    • componentWillUnmount() is invoked immediately before a component is unmounted and destroyed.
    • Perform any necessary cleanup in this method, such as invalidating timers, canceling network requests, or cleaning up any DOM elements that were created in componentDidMount.
